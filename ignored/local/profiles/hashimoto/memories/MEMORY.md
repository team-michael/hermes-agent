`user-csv-mailer` has three failure modes: (1) S3 multipart timeout >50K CSV rows (Duration ~900s); (2) **DynamoDB throttling** when `user_property_fields` table ReadCapacity=1 RCU + Lambda batch_size=10 concurrency (Duration ~30ms, "No such project" ERROR). Fix: increase RCU from 1→5-10 in `infra/terraform/prod/ap-northeast-2/dynamodb/tables.tf`. Workaround: exponential backoff in `lib/ddb.ts:getUserPropertyHeaders()` catching `ProvisionedThroughputExceededException`; (3) SQS DLQ with maxReceiveCount=1. Always verify DynamoDB RCU before assuming code bug.
