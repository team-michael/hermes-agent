# App Concept Translation Pitfalls from TikTok Niche Research

Use after `adhoc-niche-concept-research.md` when the user asks to turn a discovered TikTok concept into app creative, not just report the research.

## Preserve the original viral mechanic

If a niche cluster wins because of a specific mechanic, the first creative pass must keep that mechanic as the promise of the content.

Example: **date ideas** works because it is a save/tag listicle. A user correction in-session: a pure "date-ending ritual" concept was interesting, but it stopped being a date-ideas video. The correct translation is:

- **Wrong** — "we end every date by writing one line in the app". This is a ritual explainer, not date ideas.
- **Right** — "5 date ideas we actually did — with what we each wrote after". This keeps the date-ideas list/save mechanic, while the app ritual becomes proof and differentiation.

General rule:

1. Name the niche mechanic: save list, debrief, challenge, transformation, test, POV, storytime, etc.
2. Make the hook promise that mechanic first.
3. Place the app as the container, proof layer, or payoff — not as a detour from the mechanic.
4. If the user says the concept drifted, reframe by moving the product behavior into the **signature detail** and restoring the original content promise.

## Product as proof, not interruption

For app-growth creatives, app insertion is strongest when it makes the trend feel more real:

- date ideas → app stores the tried ideas + his/her one-line review
- debrief → app provides the receipts/log/history
- challenge → app randomizes or tracks the challenge
- transformation → app shows before/after archive
- listicle → app is the saved list or checklist

The audience should feel: "this format is better because the app exists," not "the format stopped so the ad could begin."
