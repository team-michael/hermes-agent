Free-form change requests also allowed in the pre-email thread. Report generation runs locally (not CI) because of TikTok/Google CAPTCHA and crawling.
§
Clix weekly report delivery mechanics: GitHub Release tag pattern `report-{app-id}-{week}` (e.g. report-candle-2026-W19) with a single .zip asset holding report.json, report.md, thumbnails.json, thumbs/. publish-report workflow downloads that zip, publishes to just-went-viral-web, promotes the release, and sends email. Common failure: release tag missing or app_id mismatch — rebuild via `bash scripts/upload_report.sh /tmp/clix_p2_{app}_{runid} {app-id} {week}` then re-run the workflow.
